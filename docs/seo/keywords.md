## 4. Keywords Informacionales vs. Transaccionales

### Informacionales (blog, tutoriales, guías)
Objetivo: captar tráfico orgánico, posicionar como autoridad.

- qué es vibe coding
- cómo funciona programar con IA
- diferencia entre Cursor y Claude Code
- cuáles son las mejores herramientas para vibe coding
- para qué sirve Claude Code
- cómo elegir un proyecto de IA para principiantes
- qué es un agente de IA
- cuánto tiempo tarda hacer una app con IA

**Estrategia**: Usar en páginas de detalle de proyectos (`/proyectos/[slug]`) y futuro `/blog`.

### Transaccionales (con intención de hacer/crear algo)
Objetivo: captar usuarios listos para construir un proyecto.

- crear [tipo de proyecto] con IA paso a paso
- tutorial completo [herramienta] 2025
- cómo hacer [app/bot/agente] con IA desde cero
- descargar template [proyecto] con IA

**Estrategia**: Usar en la homepage (CTA secciones) y en el hero de cada proyecto.

## 6. Estrategia de Distribución de Keywords

### Homepage `/`
- **H1**: keyword primaria + propuesta de valor
  - Ejemplo: "Ideas de proyectos para aprender a programar con IA"
- **Meta title**: "Vibe Coding — Ideas de proyectos para programar con IA"
- **Meta description**: incluir "vibe coding", "proyectos con IA", "principiante / avanzado"
- **Body**: "vibe coding", "programar con IA", nombres de herramientas (Claude Code, Cursor)

### Listing de Proyectos `/proyectos`
- **H1**: "Proyectos de Vibe Coding"
- Keywords de nivel (principiante, intermedio, avanzado) en los filtros y labels
- Keywords de categoría en los chips/tags de cada card

### Páginas de Proyecto `/proyectos/[slug]`
- **H1**: Nombre del proyecto con keyword descriptiva
- **H2/H3**: Keywords long-tail del nivel + categoría del proyecto
- Herramientas mencionadas en el frontmatter → texto ancla natural
- Meta description: beneficio + nivel + tecnología principal

### About `/sobre`
- Keywords de marca: "vibeiscoding", "comunidad vibe coding español"
- Mencionar herramientas y audiencia para señales contextuales

### Futuro `/blog`
- Keywords informacionales long-tail
- Cada artículo ataca una keyword específica + variantes semánticas

### Futuro `/categorias/[categoria]`
- Keywords de categoría como slug y H1
- Ej: `/categorias/automatizacion` → "proyectos de automatización con IA"

### Futuro `/nivel/[nivel]`
- Keywords de nivel como slug y H1
- Ej: `/nivel/principiante` → "proyectos de vibe coding para principiantes"
