## El Manifiesto

### Párrafo 1 — El problema que vimos
```
Durante años, la brecha entre tener una idea y poder ejecutarla fue el mayor filtro del desarrollo de software. Necesitabas equipo, tiempo, stack maduro, y meses de curva de aprendizaje antes de ver algo corriendo en producción. Ese modelo roto ya tiene fecha de vencimiento.
```

### Párrafo 2 — Qué es el Vibe Coding
```
Vibe Coding es desarrollar con IA como co-pilot permanente. No es usar ChatGPT para corregir un bug. Es pensar en términos de producto, describir lo que querés construir, iterar en tiempo real con herramientas como Claude Code, Cursor u OpenCode, y llegar a producción en horas en lugar de semanas. El foco se desplaza: de escribir código a dirigirlo.
```

### Párrafo 3 — Por qué existe este sitio
```
Vibe Is Coding existe porque la mayor barrera ya no es técnica — es la falta de ideas concretas para empezar. Tenemos el directorio: proyectos reales, organizados por nivel de dificultad, con stack sugerido y pasos claros. No tutoriales teóricos. Ideas que podés empezar hoy, con las herramientas que ya tenés.
```

### Párrafo 4 — A quién está dirigido
```
Si sos developer y todavía no probaste construir algo completo con IA de principio a fin, este es tu punto de entrada. Si ya vibeás y querés ideas más complejas para explorar, también. Y si venís del lado del producto — sin background técnico profundo pero con curiosidad y ganas de hacer — bienvenido. El vibe coding es, en parte, eso: bajar el piso de entrada sin comprometer el techo.
```

## Valores / Pilares

> Los nombres de los pilares se escriben en inglés — mantienen el tono del design system y suenan como categorías de sistema, no como frases de cartel motivacional. El cuerpo va en español.

### Pilar 1
**Nombre**: `ITERACIÓN BRUTAL`
**Descripción**:
```
El MVP no es el destino — es el método. Creemos en lanzar rápido, medir en contexto real y mejorar con evidencia. No en planificar en el vacío hasta que la idea pierda energía.
```

### Pilar 2
**Nombre**: `HERRAMIENTAS CON CRITERIO`
**Descripción**:
```
No recomendamos stack por hype. Cada sugerencia de herramienta en este sitio tiene una razón: costo, velocidad de setup, compatibilidad con flujos de IA, comunidad activa. Si cambiamos de opinión, lo decimos.
```

### Pilar 3
**Nombre**: `CLARIDAD TÉCNICA`
**Descripción**:
```
El vibe coding no es magia ni atajo. Es un cambio de flujo de trabajo que requiere entender lo que estás haciendo, aunque no escribas cada línea. Acá no simplificamos en exceso ni inflamos lo básico — hablamos de developer a developer.
```

### Pilar 4
**Nombre**: `ACCESO ABIERTO`
**Descripción**:
```
Todo el contenido del sitio es gratuito. Siempre. No hay paywall en las ideas. El conocimiento sobre cómo construir cosas debería estar disponible para quien quiera aprender — sin importar dónde está parado cuando llega.
```
